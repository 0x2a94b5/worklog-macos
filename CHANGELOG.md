# Changelog

本项目遵循 [Semantic Versioning](https://semver.org/)。

## [1.0.2] - 2026-07-15

### Added

- 数据库打开失败时进入恢复界面，可选择经过完整性校验的内部备份进行恢复
- 恢复前保留故障数据库及 WAL/SHM 文件，避免覆盖后失去进一步排查能力
- 增加隔离数据目录的 UI 交互测试，覆盖焦点、末尾光标、Return 保存、Escape 取消和点击外部保存
- 增加构建后启动冒烟测试，防止编译成功但应用启动即退

### Changed

- 任务进入编辑时由底层文本框可靠获得焦点并将光标放在标题末尾，同时保留用户后续点击定位光标的行为
- 月度进度统一按一级任务统计，父任务行显示已完成与全部子任务数量
- 父子任务状态保持一致：切换父任务同步全部子任务，子任务变化自动重算父任务
- 月度复盘按一级任务计算完成率，并在界面与复制内容中保留子任务进度和层级
- 滚动条沿用 macOS 原生样式，降低自定义绘制带来的兼容性风险
- 每日备份改为后台执行，并在跨天、系统唤醒和应用重新激活时检查，适配长期常开

### Fixed

- 修复 macOS 12 下滚动条持续显示的问题，滚动时显示、空闲时淡出，并尊重系统“始终显示”设置
- 降低原生滚动条显示时的不透明度，使列表滚动反馈更轻量、减少视觉干扰
- 月度复盘完整展示并复制全部主任务和子任务，支持长标题滚动查看
- 月度复盘增加关闭、重试和复制结果反馈，并使用实际完成情况生成结论
- 月度复盘改为紧凑 Sheet，使用线性进度和状态分段切换，避免面板大于主窗口
- 月度数据读取失败时不再使用当前筛选列表兜底，避免生成错误月份报告
- 修复替换 SwiftUI 内部滚动条导致应用启动时类型转换崩溃的问题
- 数据库初始化失败不再直接触发 `fatalError` 退出

## [1.0.1] - 2026-07-14

### Fixed

- 收紧任务行左侧缩进并优化状态、标题、分类和拖动控件的对齐
- 长标题最多显示两行，并可悬停查看完整内容
- 修复长标题编辑时输入区域被挤压和焦点不稳定的问题，编辑态标题独占第一行宽度
- 标题与分类均支持 Return 保存、Escape 取消和失焦保存
- 修复窄窗口下分类标签文字被完全压缩的问题，并按实际文字宽度自适应标签尺寸
- 优化选中任务的分类标签对比度、任务标题字重和快速添加提示
- 移除任务列表的大面积系统焦点环，同时保留方向键、删除、完成切换和编辑快捷操作
- 行内编辑支持点击列表空白或切换到其他界面自动保存，空标题仍保持编辑状态，内容未变化时不刷新更新时间
- 修复点击标题或分类输入框末尾被误判为外部点击并提前保存的问题
- 取消基于标题关键词的隐藏分类推断；新增任务默认未分类，分类视图新增继承当前分类，子任务继承主任务
- 快速新增仅在保存成功后清空内容，并在连续录入时保持输入焦点
- 空标题不再静默退出编辑，清空分类输入后统一归入未分类

## [1.0.0] - 2026-07-14

### Added

- 原生 SwiftUI 与 AppKit macOS 应用，最低支持 macOS 12
- 月份、分类及未完成/已完成状态筛选
- 主任务与子任务、折叠和同级拖动排序
- 单击选择、双击编辑及键盘导航
- 全局搜索与任务定位
- Markdown 追加导入和标准 `.md` 文件导出
- SQLite 本地存储、版本迁移、事务和滚动备份
- 删除后的会话级撤销恢复
- 月度复盘生成与复制
- 核心回归测试和 GitHub Actions 构建检查
